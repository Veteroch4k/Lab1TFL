# 3. Постановка задачи и описание проекта
Данный проект представляет собой графический интерфейс (GUI) разрабатываемого компилятора, объединенный с модулем лексического анализа.
Приложение реализует полноценный многовкладочный текстовый редактор кода (подготовка из Лабораторной работы 1) и конечный автомат (FSM), который разбивает введенный код на токены (лексемы) и отлавливает лексические ошибки.

**Общие требования к сканеру:**
* Вынесение логики анализа в отдельный модуль.
* Поддержка многострочности (корректный подсчет строк и столбцов при переносах).
* Вывод результатов в графический интерфейс (таблица лексем и таблица ошибок).
* Навигация по коду при клике на найденную лексему/ошибку.

# 5. Диаграмма состояний (Теоретическая модель FSM)

![Диаграмма состояний](LABA 2 TUFK.png)  
*Рисунок 1 — Конечный автомат лексического анализатора*

**Краткое описание работы автомата:**
* **START (0):** Начальное состояние. Читает текущий символ и определяет ветку логики. Игнорирует пробелы, символы табуляции и переносы строк (с увеличением счетчика).
* **Сборка слов (1):** При чтении буквы уходит в цикл сбора букв и цифр. По завершении сверяет слово со словарем (коды 1-23). Если совпадений нет — выдает идентификатор (код 24).
* **Одиночные символы (2-11, 21):** Прямые переходы для пунктуации и операторов прототипа функции. Возвращают соответствующие коды сразу после прочтения.
* **Переменные (12-13):** Переход по символу `$`, после чего собираются буквы, цифры и подчеркивания (код 36).
* **Строки и числа (14-20):** Обработка кавычек с циклом до закрывающего символа (коды 37, 38) и цифр, включая дробные числа с плавающей точкой (коды 39, 40). Поддерживается многострочность внутри строковых литералов.
* **ERROR:** Фиксируется, если встречен недопустимый символ.

# 7. Инструкция по сборке и запуску
Для работы приложения необходим установленный JDK версии не ниже 21 и система сборки Maven.
1. Склонируйте репозиторий с проектом в удобную директорию и откройте в IntelliJ IDEA.
2. Среда разработки автоматически подтянет зависимости из `pom.xml`.
3. Откройте терминал в корневой папке проекта и выполните команду для очистки и компиляции:
   ```bash
   mvn clean compile
   ```
4. Для запуска приложения используйте команду:
   ```bash
   mvn javafx:run
   ```
*Главный класс располагается по пути: `src/main/java/org/veteroch4k/laba1/HelloApplication.java`.*

# 8. Описание интерфейса и функций (Руководство пользователя)

![Главное окно приложения](img1.png)  
*Рисунок 2 — Главное окно приложения*

**Работа с текстом и файлами:**
* Поддерживается многовкладочный интерфейс с одновременным открытием нескольких файлов.
* Создание (**Ctrl+N**), Открытие (**Ctrl+O**), Сохранение (**Ctrl+S**).
* Поддержка **Drag-and-Drop** для файлов из ОС.
* Автоматическая нумерация строк и масштабирование текста (**Ctrl + Колесико мыши**).
* Динамическая строка состояния с позицией курсора.

**Лексический анализ:**
* Запуск сканера осуществляется кнопкой **"Пуск"** на панели инструментов или клавишей **F5**.
* В нижней панели расположены две таблицы:
   * **Таблица "Лексемы":** Содержит корректно распознанные элементы (тип, значение, строка, столбец).
   * **Таблица "Ошибки (Analyzer)":** Выводит нераспознанные символы и незакрытые строки.
* **Интерактивная навигация:** При клике на любую строку в таблицах лексем или ошибок курсор в верхнем редакторе автоматически переходит к проблемному символу и выделяет его.
