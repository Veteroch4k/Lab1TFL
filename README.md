# 6. Синтаксический анализатор (Парсер)

### 6.1 Разработка грамматики $G[Prototype]$
Грамматика описывает правила построения прототипа функции в PHP:

1. `<Prototype>  ::= KW_FUNCTION IDENTIFIER L_PAREN <Args> R_PAREN <RetType> SEMICOLON`
2. `<Args>       ::= <Arg> <ArgsTail> | ε`
3. `<ArgsTail>   ::= COMMA <Arg> <ArgsTail> | ε`
4. `<Arg>        ::= <Type> VARIABLE <DefaultVal>`
5. `<Type>       ::= QUESTION TYPE_NAME | TYPE_NAME | ε`
6. `<DefaultVal> ::= ASSIGN <Literal> | ε`
7. `<Literal>    ::= NUMBER | FLOAT | STRING`
8. `<RetType>    ::= COLON <Type> | ε`

*(где `ε` означает пустое правило / отсутствие элемента)*

### 6.2 Классификация грамматики
По иерархии Хомского данная грамматика является **контекстно-свободной (КС-грамматикой)**.
Она относится к классу **LL(1)**, так как правила леворекурсивны (отсутствуют), однозначны, и для выбора альтернативы при разборе парсеру достаточно предпросмотра одного следующего токена.

### 6.3 Метод анализа
Используется алгоритм **рекурсивного спуска**. Для каждого нетерминала грамматики реализован отдельный метод (например, `parsePrototype()`, `parseArgs()`, `parseType()`). Парсер последовательно вызывает эти методы, "поглощая" ожидаемые токены из списка, переданного сканером.

### 6.4 Диагностика и нейтрализация ошибок (Метод Айронса)
Для предотвращения остановки анализатора при первой же ошибке реализована нейтрализация на уровне фразы (упрощенный метод Айронса).
**Алгоритм нейтрализации:**
1. Если встречен неожиданный токен, фиксируется сообщение об ошибке с указанием позиции.
2. Анализатор "пропускает" ошибочные токены, пока не встретит ожидаемый токен или один из **синхронизирующих символов** (например, скобки `(`, `)` или точку с запятой `;`), с учетом FOLLOW-множеств для предотвращения каскадного обрушения.
3. Это предотвращает появление ложных ошибок и позволяет продолжить анализ оставшейся части строки.

### 3. Корректная строка со всеми необязательными элементами
**Код:** `function myFunc(int $amount, ?string $currency = 'USD'): float;`
* **Описание:** Проверка работы эпсилон-правил грамматики: парсинг nullable-типа (`?string`), значения по умолчанию (`= 'USD'`) и возвращаемого типа функции (`: float`). Ошибок не найдено.

![Скриншот теста 3](test3.png)

### 4. Строка с синтаксической ошибкой (Отработка метода Айронса)
**Код:** `function 123($a ;`
* **Описание:** Вместо имени функции введено число `123`, а также пропущена закрывающая скобка `)`. Метод Айронса успешно изолировал число, не нарушив дальнейший разбор, а затем виртуально подставил пропущенную скобку перед точкой с запятой.
* **Ошибки в таблице:** 1. `Ожидалось имя функции. Встречено: '123'`
   2. `Ожидалась закрывающая скобка ')'. Встречено: ';'`

![Скриншот теста 4](test4.png)

### 5. Ошибка возвращаемого типа (Граничный случай с эпсилон-правилом)
**Код:** `function getData(): ;`
* **Описание:** Программист поставил двоеточие для указания возвращаемого типа, но забыл написать сам тип данных. Так как двоеточие присутствует, парсер строго требует наличие типа данных и не может уйти в эпсилон-правило.
* **Ошибка в таблице:** `Синтаксическая ошибка: Ожидался тип данных (int, string и т.д.). Встречено: ';'`

![Скриншот теста 5](test5.png)

### 6. Эпсилон-пропуск разделителя (Отсутствие двоеточия)
**Код:** `function test(int $a) float;`
* **Описание:** Демонстрация корректной логики LL(1) парсера. Пропущено двоеточие `:` перед типом `float`. Парсер применяет эпсилон-правило, решая, что тип опущен, и переходит к ожиданию точки с запятой `;`. Встретив `float` вместо `;`, парсер выдает обоснованную ошибку.
* **Ошибка в таблице:** `Ожидалась точка с запятой ';' в конце прототипа. Встречено: 'float'`

![Скриншот теста 6](test6.png)

### 7. Синтаксическая «каша» (Защита от каскадного обрушения)
**Код:** `4324234f342unctfsdfion myFunc(int $amount): float;`
* **Описание:** Проверка работы FOLLOW-множеств при нейтрализации. Анализатор игнорирует число `4324234`, берет первый попавшийся идентификатор `f342unctfsdfion` как имя функции, а затем отбрасывает реальное имя `myFunc`, чтобы восстановить логику разбора на скобке `(`.
* **Ошибки в таблице:**
   1. `Ожидалось ключевое слово 'function'. Встречено: '4324234'`
   2. `Ожидалась открывающая скобка '('. Встречено: 'myFunc'`
